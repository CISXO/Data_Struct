#Stack

1. 스택, 큐, 덱이란?

```-데이터 값을 저장하는 기본적인 구조로 일차원의 선형 자료구조이다.

	-배열/리스트와 유사하게 값을 저장하는 연산과 저장된 값을 꺼내는 연산이 제공된다.
```
 a) 스택(Stack): LIFO - 후입 선출
```
	-스택은 가장 최근에 저장된 값 다음에 저장되며, 가장 최근에 저장된 값이 먼저 나간다. LIFO - last in first out
```
 b) 큐 (Queue): FIFO - 선입 선출
```
	- 큐는 스택과 동일하게 가장 최근에 저장된 값 다음에 새로운 값이 저장되지만 가장 오래전에 저장된 값부터 나간다. FIFO - first in first out

       - 스택과 마찬가지로 리스트는 큐의 모든 연산을 지원하지만, 리스트는 동적 배열로 구현되어 큐의 연산을 수행하기엔 효율적이지 않다.
       - que, max, front, rear, num<br>
         que: 인큐하는 데이터를 저장하기 위한 배열 <br>
          max: 큐의 최대 용량을 저장하는 필드로 이 값은 배열 que에 저장할 수 있는 <br>
         최대 요소의 개수와 같다.<br>
         front:인큐하는 데이터 가운데 첫 번째 요소의 인덱스를 저장하는 필드<br>
         rear: 인큐한 데이터 가운데 맨 나중에 넣은 요소의 하나 뒤의 인덱스를 저장하는 필드<br>
          num: 큐에 쌓아 놓은 데이터 수<br>
```
 3) 덱 (Dequeue): Stack + Queue 
```
       - 덱은 스택과 큐의 연산을 모두 지원하는 자료구조이다.

       - 왼쪽과 오른쪽에서 모두 삽입과 삭제가 가능한 큐다.
```
 2. java stack
```
       - 검색 메서드 (indexOf) 스택 본체의 배열에 포함되어 있는지 포함되어 있지 않은지 조사하는 메서드

       - 스택의 모든 요소를 삭제(cler): 스택의 모든 데이터 삭제

       - 용량을 확인하는 메서드 (capacity): 스택의 max 값을 그대로 반환

       - 데이터 수를 확인하는 메서드 (size): 데이터 개수 반환 

       - 스택이 비어있는지 검사하는 메서드 (isEmpty): 비어있으면 true, 그렇지 않으면 false

       - 스택이 가득 찼는지 검사하는 메서드(isFull): 가득 찼으면 true, 그렇지 않으면 false
```
 3. java queue
```
       - 피크 메서드 (peek): 맨 앞의 데이터를 보는 메서드 데이터를 꺼내지 않음 <br>
  데이터가 비어있으면 예외처리

       - 검색 메서드 (indexOf): 큐의 배열에서 x와 같은 데이터가 저장되어 있는 위치를<br>
 를 알아내는 메서드 실패하면 -1을 반환 

       - 모든 데이터를 삭제하는 메서드(clear): 큐의 모든 데이터를 삭제하는 메서드

       - 최대 용량을 확인하는 메서드(capacity): 큐의 최대 용량을 확인하는 메서드

       - 데이터 수를 확인하는 메서드(size): 현재 큐의 데이터 수를 반환하는 메서드

       - 큐가 비어있는지 판단하는 메서드(isEmpty): 비어 있으면 true, 그렇지 않으면 false

       - 큐가 가득 찼는지 판단하는 메서드(isFull): 가득 차있으면 true, 그렇지 않으면 false

       - 모든 데이터를 출력하는 메서드 (dump): 큐에 인큐된 모든 데이터를 프런트에서 리어<br>
       순으로 출력하는 메서드 큐가 비어있으면 '큐가 비어있습니다'
```
